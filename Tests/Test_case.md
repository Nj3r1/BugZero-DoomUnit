## Non-Functional Testing

| Test Case ID | Feature | Test Scenario | Steps to Reproduce | Expected Result | Actual Result | Status | Notes |
|---|---|---|---|---|---|---|---|
| NFT_001 | Performance (Load Times) | Page load time for homepage | 1. Open the application's homepage in a browser. <br> 2. Use browser developer tools (Network tab) to measure page load time. | Homepage loads within an acceptable timeframe (e.g., < 3 seconds). | | | |
| NFT_002 | Performance (Responsiveness) | Application responsiveness under load | 1. Use a load testing tool (e.g., JMeter, LoadRunner) to simulate 50 concurrent users accessing various parts of the application (registration, login, scheduling). | The application remains responsive with acceptable response times (e.g., < 2 seconds for API calls, < 5 seconds for page loads) and no significant errors. | | | |
| NFT_003 | Security (Data Validation) | SQL Injection prevention | 1. In any input field (e.g., search bar, login field), enter known SQL injection payloads (e.g., `' OR '1'='1`). | The application should prevent SQL injection attacks by sanitizing input or using prepared statements, and not reveal sensitive data or throw database errors. | | | |
| NFT_004 | Security (XSS Prevention) | Cross-Site Scripting (XSS) prevention | 1. In any input field that displays user-generated content (e.g., comment section, profile bio), enter known XSS payloads (e.g., `<script>alert('XSS')</script>`). | The application should sanitize or encode user input to prevent XSS attacks, and the script should not execute. | | | |
| NFT_005 | Usability (User Experience) | Intuitive navigation | 1. Navigate through the application's main features (registration, login, scheduling, blog). | Navigation is clear, logical, and easy to understand for a new user. | | | |
| NFT_006 | Usability (User Experience) | Clear error messages | 1. Intentionally trigger various errors (e.g., invalid input, network issues). | Error messages are clear, concise, and provide helpful guidance to the user. | | | |
| NFT_007 | Accessibility (Screen Readers) | Navigation with a screen reader | 1. Enable a screen reader (e.g., NVDA, JAWS). <br> 2. Navigate through key pages and features using only keyboard and screen reader. | All interactive elements are correctly identified and announced by the screen reader. | | | |
| NFT_008 | Accessibility (Keyboard Navigation) | Keyboard-only navigation | 1. Disable mouse and navigate through the entire application using only the keyboard (Tab, Shift+Tab, Enter, Spacebar). | All interactive elements are reachable and operable using only the keyboard, and focus is clearly indicated. | | | |
